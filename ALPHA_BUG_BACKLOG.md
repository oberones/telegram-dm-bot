# Alpha Bug Backlog

## Purpose

This file is the working backlog for Alpha findings.

Use one flat entry per issue. Keep the format compact and consistent so issues can be triaged quickly.

## Severity Guide

- `P0`: data loss, unrecoverable failure, major security issue
- `P1`: broken core flow, wrong system state, failed moderation or recovery
- `P2`: confusing UX, weak explanation, missing visibility, medium-impact bug
- `P3`: polish issue, small copy/layout annoyance, low-impact defect

## Status Guide

- `new`
- `triaged`
- `in_progress`
- `fixed`
- `deferred`
- `verified`
